Nota Bene for The Ripple Protocol Consensus Algorithm 3. Ripple Consensus Algorithm

RPCA - The Ripple Protocol consensus algorithm

Strong correctness - no fraudulent transactions will be approved AND any non-fraudulent transactions will be approved (less than 20% for RPCA)

Weak correctness - no fraudulent transactions will be approved. The consensus process will fail, but it will still not be possible to confirm a fraudulent transaction (less than 80% for RPCA).
